noc_bounty

A comprehensive bounty hunting system that brings the lawless frontier to life with dynamic wanted posters, tracking mechanics, and immersive rewards.

The noc_bounty script delivers an authentic bounty hunting experience for your RedM server. Players can track down wanted criminals, collect rewards, and build their reputation as legendary bounty hunters or notorious outlaws.

✨ Key Features
  • Dynamic bounty board with wanted posters
  • Multiple bounty tiers (Minor, Major, Legendary)
  • Realistic tracking and investigation mechanics
  • Bounty hunter license system
  • Reputation and reward scaling
  • Dead or Alive capture options

Installation

Step 1: Download and Extract

Download noc_bounty from our store and extract it to your resources folder:

resources/
└── [noc]/
    └── noc_bounty/

Step 2: Add to server.cfg

Add the following to your server.cfg. Note that noc_bounty should be loaded after any framework resources:

ensure noc_bounty

Step 3: Database Setup

The script automatically creates necessary tables on first start. Alternatively, you can manually run the SQL migration:

-- Located at: noc_bounty/sql/migration.sql
-- Creates tables: noc_bounties, noc_hunter_stats, noc_bounty_claims

Step 4: Configure

Open noc_bounty/config/config.lua to customize bounty settings, rewards, and tracking mechanics.

Configuration

General Settings

Config = {}

-- General Settings
Config.Debug = false                    -- Enable debug mode
Config.Locale = 'en'                    -- Language code
Config.RequireLicense = true            -- Require bounty hunter license

-- Bounty Board Locations
Config.Boards = {
    { coords = vector3(-275.0, 805.0, 120.0), type = 'sheriff' },
    { coords = vector3(2550.0, -1150.0, 55.0), type = 'post_office' },
}

-- Bounty Tiers
Config.Tiers = {
    minor = {
        minReward = 25,
        maxReward = 75,
        difficulty = 1,
        trackingRadius = 500.0
    },
    major = {
        minReward = 100,
        maxReward = 250,
        difficulty = 2,
        trackingRadius = 300.0
    },
    legendary = {
        minReward = 500,
        maxReward = 1000,
        difficulty = 3,
        trackingRadius = 150.0
    }
}

Tracking Settings

Configure how bounty tracking works:

Config.Tracking = {
    UpdateInterval = 30,                -- Seconds between position updates
    MaxClues = 5,                       -- Maximum clues before target reveal
    ClueRadius = 200.0,                 -- Search radius for clues
    FadeTime = 60,                      -- Seconds before trail fades
    RequireItems = true,                -- Require tracking items (map, binoculars)
}

Config.TrackingItems = {
    { name = 'map', required = true },
    { name = 'binoculars', required = false, bonus = true }
}

Dead or Alive Settings

Parameter Type Default Description
AllowDeadCapture boolean true Allow bringing in bounties dead

Bounty Types

Minor Bounties

Low-level criminals with moderate rewards. Perfect for new bounty hunters:

  • Simple tracking requirements
  • Larger search radius
  • Lower risk encounters
  • Rewards: $25 - $75

Major Bounties

Dangerous criminals requiring skill and preparation:

  • More complex tracking
  • Targets may have allies
  • Better equipment recommended
  • Rewards: $100 - $250

Legendary Bounties

The most notorious outlaws on the frontier:

  • Extremely difficult to track
  • Highly dangerous targets
  • May require a posse
  • Rewards: $500 - $1000+
⚠️ Warning

Legendary bounties are designed for experienced hunters. Make sure you're well-armed and consider bringing backup.

Tracking System

The tracking system uses a clue-based investigation approach:

Investigation Phase

  1. Accept bounty from the board
  2. Travel to the last known location
  3. Search the area for clues (footprints, witnesses, etc.)
  4. Each clue narrows down the target's location
  5. After enough clues, the target is revealed on your map

Clue Types

Clue Type Description Information Revealed
Footprints Trail left by the target Direction of travel
Witness NPC who saw the target Last known location
Camp Site Recent camp location Time since departure
Drop Point Hidden stash Target's plans

Tracking Tips

  • Bring a map and binoculars for bonus tracking abilities
  • Check with locals - they might have seen something
  • Time matters - trails go cold after a while
  • Weather affects tracking (rain washes away footprints)

API Reference

Server Exports

-- Create a new bounty
exports.noc_bounty:CreateBounty(targetId, tier, reason, reward, callback)

-- Get all active bounties
local bounties = exports.noc_bounty:GetActiveBounties()

-- Get bounty by target ID
local bounty = exports.noc_bounty:GetBountyByTarget(playerId)

-- Claim a bounty (called when hunter captures target)
exports.noc_bounty:ClaimBounty(hunterId, targetId, isAlive, callback)

-- Get hunter stats
local stats = exports.noc_bounty:GetHunterStats(playerId)

-- Check if player has bounty hunter license
local hasLicense = exports.noc_bounty:HasLicense(playerId)

Client Exports

-- Open bounty board UI
exports.noc_bounty:OpenBountyBoard()

-- Get current tracking info
local tracking = exports.noc_bounty:GetTrackingInfo()

-- Check if tracking a bounty
local isTracking = exports.noc_bounty:IsTracking()

-- Abandon current bounty
exports.noc_bounty:AbandonBounty()

-- Get nearby clues (if tracking)
local clues = exports.noc_bounty:GetNearbyClues()

Events

-- Server: When a bounty is created
TriggerEvent('noc_bounty:bountyCreated', bountyData)

-- Server: When a bounty is claimed
TriggerEvent('noc_bounty:bountyClaimed', hunterId, targetId, reward, isAlive)

-- Server: When hunter reputation changes
TriggerEvent('noc_bounty:reputationChanged', hunterId, newRep, delta)

-- Client: When tracking updates
TriggerClientEvent('noc_bounty:trackingUpdate', hunterId, trackingData)

-- Client: When new clue is found
TriggerClientEvent('noc_bounty:clueFound', hunterId, clueData)
