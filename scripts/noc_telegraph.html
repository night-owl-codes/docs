noc_telegraph

A period-accurate telegraph system for immersive long-distance communication in your RedM server.

The noc_telegraph script brings authentic 1899-era communication to your RedM server. Players can send and receive telegrams through telegraph offices, creating immersive roleplay opportunities for businesses, law enforcement, and personal correspondence.

✨ Key Features
  • Period-accurate telegraph system with Morse code aesthetics
  • Multiple telegraph office locations
  • Message delivery delays based on distance
  • Telegram logging for server administrators
  • Customizable pricing and delivery times

Installation

Step 1: Download and Extract

Download noc_telegraph from our store and extract it to your resources folder:

resources/
└── [noc]/
    └── noc_telegraph/

Step 2: Add to server.cfg

Add the following to your server.cfg:

ensure noc_telegraph

Step 3: Run Database Migration

Execute the provided SQL file to create the necessary database tables:

-- The migration runs automatically on first start
-- Or manually import: noc_telegraph/sql/migration.sql

Step 4: Configure

Open noc_telegraph/config/config.lua to customize the script settings.

Configuration

The configuration file is organized into several sections for easy customization:

General Settings

Config = {}

-- General Settings
Config.Debug = false                    -- Enable debug mode for development
Config.Locale = 'en'                    -- Language code (en, es, de, fr)
Config.CurrencySymbol = '$'             -- Currency display symbol

-- Telegram Settings
Config.BasePrice = 5                    -- Base price per telegram
Config.PricePerWord = 0.10              -- Additional cost per word
Config.MaxMessageLength = 500           -- Maximum character limit
Config.DeliveryDelay = 300              -- Base delivery time in seconds

Office Locations

Configure telegraph office locations throughout your map:

Config.Offices = {
    {
        name = "Saint Denis Telegraph",
        coords = vector3(2700.0, -1400.0, 50.0),
        hours = { open = 6, close = 22 },
        npc = {
            model = "U_M_M_ValSheriff_01",
            coords = vector4(2700.0, -1400.0, 50.0, 180.0)
        }
    },
    {
        name = "Blackwater Telegraph",
        coords = vector3(-800.0, -1300.0, 40.0),
        hours = { open = 7, close = 21 },
        npc = {
            model = "U_M_M_ValSheriff_01",
            coords = vector4(-800.0, -1300.0, 40.0, 90.0)
        }
    }
}

Usage

Sending a Telegram

Players can visit any telegraph office to send messages:

  1. Approach the telegraph clerk NPC
  2. Interact using the prompt (default: E)
  3. Select "Send Telegram" from the menu
  4. Enter the recipient's name and message
  5. Confirm the price and send

Receiving Telegrams

When a telegram arrives, the recipient will be notified:

  • A notification appears indicating a new telegram
  • Visit any telegraph office to collect your messages
  • Telegrams are stored until collected or expired

API Reference

Server Exports

-- Send a telegram programmatically
exports.noc_telegraph:SendTelegram(sender, recipient, message, callback)

-- Get pending telegrams for a player
local telegrams = exports.noc_telegraph:GetPendingTelegrams(playerId)

-- Check if player has unread telegrams
local hasUnread = exports.noc_telegraph:HasUnreadTelegrams(playerId)

-- Mark telegram as read
exports.noc_telegraph:MarkAsRead(telegramId)

-- Get telegram history
local history = exports.noc_telegraph:GetHistory(playerId, limit)

Client Exports

-- Open the telegraph UI
exports.noc_telegraph:OpenUI()

-- Close the telegraph UI
exports.noc_telegraph:CloseUI()

-- Get nearby telegraph office
local office = exports.noc_telegraph:GetNearbyOffice()

-- Check if player is at telegraph office
local atOffice = exports.noc_telegraph:IsAtOffice()

Events

The script fires the following events for integration:

-- Server Event: When a telegram is sent
TriggerEvent('noc_telegraph:telegramSent', sender, recipient, messageData)

-- Server Event: When a telegram is delivered
TriggerEvent('noc_telegraph:telegramDelivered', recipient, telegramId)

-- Client Event: When player receives notification
TriggerClientEvent('noc_telegraph:notifyRecipient', playerId, telegramCount)
