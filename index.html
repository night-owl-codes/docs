Getting Started

Welcome to Night Owl Codes documentation. Here you'll find everything you need to install, configure, and customize our RedM scripts for your server.

At Night Owl Codes, we build immersive, period-accurate, and highly optimized resources for RedM server owners who care about quality. This documentation will guide you through the installation process, configuration options, and advanced features of our scripts.

💡 New to Night Owl Codes?

If you're setting up our scripts for the first time, we recommend following the installation guide below. If you hit a snag, open a ticket on our Discord and we'll help you out.

Installation

Installing Night Owl Codes scripts is straightforward. Follow these steps to get up and running:

Prerequisites

Before you begin, make sure you have the following:

  • A RedM server running on the latest version
  • Access to your server's resources folder
  • Basic knowledge of server.cfg configuration

Step 1: Download the Script

Download your purchased script from our store. The download will be a ZIP file containing all necessary resources.

Step 2: Extract to Resources

Extract the downloaded ZIP file to your server's resources folder. Each script uses the noc_ prefix for easy identification:

resources/
├── [noc]/
│   ├── noc_telegraph/
│   ├── noc_notify/
│   └── ...

Step 3: Add to server.cfg

Add the script to your server.cfg file. Make sure to place it after any framework resources:

ensure noc_telegraph
ensure noc_notify
⚠️ Important

All Night Owl Codes scripts use the noc_ prefix. If you're having issues, double-check that the resource name matches exactly in your server.cfg.

Step 4: Configure the Script

Each script comes with a configuration file located in the config folder. Open the config.lua file to customize the script to your needs:

-- config.lua
Config = {}

-- General Settings
Config.Debug = false
Config.Locale = 'en'

-- Feature toggles
Config.EnableLogging = true

Step 5: Restart Your Server

Restart your server or use the refresh command followed by the resource name to load the new script:

refresh
ensure noc_telegraph

Configuration

Each Night Owl Codes script comes with extensive configuration options. The configuration files are heavily commented to help you understand each setting.

Configuration Structure

The typical configuration file structure is as follows:

File Purpose
config.lua Main configuration settings
locales/*.lua Language/translation files
items.lua Item definitions (if applicable)

Best Practices

  • Always back up your configuration files before making changes
  • Use the provided default values as a starting point
  • Test changes in a development environment first
  • Keep your configuration files organized and well-commented

API Reference

Night Owl Codes scripts expose various API endpoints for integration with other resources. Here's a quick overview of the available exports:

Server Exports

-- Example: Check if player has a specific condition
exports.noc_telegraph:HasCondition(playerId, conditionName)

-- Example: Trigger a custom event
exports.noc_telegraph:TriggerEvent(playerId, eventData)

Client Exports

-- Example: Get current player status
local status = exports.noc_telegraph:GetStatus()

-- Example: Open UI
exports.noc_telegraph:OpenUI()

For detailed API documentation for each script, check the individual script pages in the sidebar.

Troubleshooting

Running into issues? Here are some common problems and their solutions:

Script Not Loading

If the script isn't loading, check the following:

  • Verify the resource name in server.cfg matches the folder name exactly
  • Check your server console for error messages
  • Ensure all dependencies are installed and loaded before the script

Database Errors

If you're seeing database-related errors:

  • Verify your database connection settings
  • Run any provided SQL migration files
  • Check that required tables exist

Performance Issues

If you experience performance problems:

  • Enable the debug mode to identify slow operations
  • Review your configuration for resource-intensive settings
  • Consider reducing update frequencies in the config
Need More Help?

If you're still having trouble, join our Discord and open a support ticket. We're here to help.

Frequently Asked Questions

What frameworks do you support?

Our scripts are designed to work with popular RedM frameworks. Check individual script documentation for specific compatibility information.

Can I modify the scripts?

Yes, you can modify the scripts for your own server. However, we recommend keeping the original files as a backup and documenting your changes for future updates.

How do I get updates?

Updates are delivered through our Discord. You'll receive notifications when new versions are available, and you can download them from the store using your purchase link.

Do you offer refunds?

Due to the digital nature of our products, we generally don't offer refunds. However, if you're experiencing technical issues, we'll work with you to resolve them through our support channels.
